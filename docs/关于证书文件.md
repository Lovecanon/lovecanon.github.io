 # 关于证书文件
 在开发时，难免会遇到`.pem`、`.cer`、`.p12`之类的文件，每次看到都很晕，遂写一篇博客记录下细节。

## ASN.1（Abstract Syntax Notation One) 
ASN.1是标准的接口定义语言，用于定义数据结构，然后通过编码规则（ASN.1 Encoding Rules）将数据序列化/反序列化为二进制流，进而实现两个实体之间的数据交换。

试想，如果要你来设计`数据结构`，你需要准备什么？
1. 基本类型，如：int、bool、string、double等
2. 基础数据结构，如：list、set、dict、queue等。当然也可以通过基本类型+语法规则自己构造这些基础数据结构
3. 语法规则，包括类型定义、变量声明及初始化等
   * `Age ::= INTEGER`，定义一个Age类型，它其实是INTEGER类型的别称
   * `pi REAL ::= 3.1416`，定义REAL类型的变量pi，值等于3.1416
   * `Age ::= INTEGER (0..150)`，定义一个Age类型，它其实是INTEGER类型，范围在0到150之间

不错，ASN.1就是这么做的，首先它定义了如下的基础类型：
* BOOLEAN
* INTEGER
* ENUMERATED
* REAL
* NULL
* DATE
* STRING
  * BIT STRING
  * OCTET STRING
  * IA5String（international alphabet number 5）
  * VisibleString

其次，它也定义了基础的数据结构：
* SEQUENCE
* SEQUENCE OF
* SET
* CHOICE

我就可以通过它给出的基础数据结构，定义结构体、数组等
```
// 定义CAR类型的结构体
CAR ::= SEQUENCE {
    brand IA5String
    color ENUMERATED {red, blue, grey}
    age INTEGER
}

// 定义Car类型的数组
myCars ::= SEQUENCE OF Car
```

### ASN.1 Encoding Rules
仅仅通过ASN.1定义数据结构还不行，你还需要将数据放到指定的位置上，并对其进行序列化和反序列化。ASN.1提供了几种标准的编码规则：
1. BER（Basic Encoding Rules）
2. PER（Basic Packed Encoding Rules）
3. XER（Basic XML Encoding Rules）

看到这几种编码好像有点复杂，其实不然，它和我们实际编程类似。在HTTP API接口返回数据时，我们往往先定义类，然后创建类的实例，并对各个字段填充数据，然后选择一种常用的序列化（JSON、XML等），然后返回给前端。上面说的ASN.1编码规则，和我们常用的序列化方法一样，用于编码数据以方便数据在介质上的传输。
