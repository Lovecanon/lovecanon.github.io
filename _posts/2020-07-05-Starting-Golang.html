应用程序入口：
  • 必须是 main 包：package main  // 包，表明代码所在的模块（包）
  • 必须是 main 方法：func main()
  • 文件名不一定是 main.go

退出返回值：
  • 与其他主要编程语言的差异：
    • Go 中 main函数不支持任何返回值。通过 os.Exit 来返回状态
    • Go 中 main函数不支持传入参数，func main(arg []string)。通过 os.Args 获取命令行参数

类型转换：
  • 与其他主要编程语言的差异：
    • Go语言不允许隐式类型转换
    • 自定义别名（MyType）和原有类型也不能进行隐式类型转换

指针类型
  • 与其他主要编程语言的差异：
    • 不支持指针运算
    • string 是值类型，默认的初始化值为空字符串，而不是nil
循环
  • Go语言仅支持 for 循环关键字

    • 无限循环：for {...}，
if 条件

  • 支持变量赋值：
    • 如：if declaration; condition {...}

map 声明、元素访问和遍历
package main
import "fmt"

func main() {
   // map 的声明
   // 方法一：先声明后初始化，注意后面的花括弧
   var m1 = map[string]int{}
   m1["a"] = 1

   // 方法二：声明时初始化
   var m2 = map[string]int{"a": 1, "b": 2}
   fmt.Println(m2)

   // 方法三：使用make声明map，map是可以动态增长的
   var m3 = make(map[string]int, 3)
   m3["a"] = 1
   m3["b"] = 2
   m3["c"] = 3
   m3["d"] = 4
   fmt.Println(m3)  // map[a:1 b:2 c:3 d:4]

   // map 元素访问
   if v, ok := m3["a"]; ok {
      fmt.Printf("Key is existed, value: %v", v)
   } else {
      fmt.Printf("Key is not existing")
   }

   // map 遍历
   for k, v := range m3 {
      fmt.Printf("%v: %v", k, v)
   }
}

go语言面向对象编程

Is Go an object-oriented language?
Yes and no. Although Go has types and methods and allows an object-oriented style of programming, there is no type hierarchy. The concept of “interface” in Go provides a different approach that we believe is easy to use and in some ways more general. There are also ways to embed types in other types to provide something analogous—but not identical—to subclassing. Moreover, methods in Go are more general than in C++ or Java: they can be defined for any sort of data, even built-in types such as plain, “unboxed” integers. They are not restricted to structs (classes).
Also, the lack of a type hierarchy makes “objects” in Go feel much more lightweight than in languages such as C++ or Java.

type Employee struct {
   id string
   Name string
   Age int
}

func main() {
   e := Employee{"0", "Bob", 21}
   e1 := Employee{Name:"Mike", Age: 30}
   e2 := new(Employee)  // 返回指针
   e2.id = "2"
   e2.Name = "Rose"
   e2.Age = 23

   fmt.Println(e)  // {0 Bob 21}
   fmt.Println(e1)  // { Mike 30}
   fmt.Println(e1.id)  // 空字符串

   fmt.Println(e2)  // &{2 Rose 23}
   fmt.Printf("e is %T\n", e)  // e is main.Employee
   fmt.Printf("e2 is %T\n", e2)  // e2 is *main.Employee，引用/指针类型
}

行为（方法）定义：
  • 第一种定义方式在实例对应方法被调用时，实例的成员会进行值复制
    • func (e Employee) String() string {...}
  • 通常情况下为了避免内存拷贝，我们使用第二种定义方式
    • func (e *Employee) String() string {...}
注：通过指针也可以像通过实例那样，通过 . 符号访问field。同理，也可以通过指针访问第一种定义方法方式。如：new(Employee).String()

扩展与复用
面向对象里面最有争议，也是最复杂的一部分了，就是扩展。我们知道面向对象的扩展，通常可以通过继承和复合来进行。go其实是不支持继承的，而继承是面向对象的主要特性之一，go到底是不是面向对象的也是很有争议的。

  • 和继承（extends）不一样，复合需要自己定义方法，无法通过继承父类的方法；

不一样的接口类型，一样的多态
  • 空接口：空接口可以表示任何类型，有点像C语言的 void*，Java中的Object类型
  • 断言：空接口go并不是通过强制类型转换的，通过断言来将空接口转换为指定类型
    • v, ok := p.(int)  // ok = true 时转换成功

Go接口最佳实践：
  • 倾向使用小的接口定义，很多接口只包含一个方法
  • 较大的接口定义，可以由很多小接口定义组合而成
  • 只依赖于必要功能的最小接口
